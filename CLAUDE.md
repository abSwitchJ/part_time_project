# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## 用户与目标(读完这一节再做任何事)

**用户**:大三在校,正在找 **AI 产品经理实习**。这个仓库是用户的**产品作品集**,**不是要交付上线的代码**。

**长期目标**:把仓库打磨到"敢主动甩链接给 AI PM 实习招聘方看"的程度。

**所有建议、修改、补充内容,都必须先回答这个问题**:"这一步对用户拿到 AI PM 实习的边际增益是什么,投入产出比怎样?" 不能回答清楚的事,先别做。

**严禁**:把这个项目当工程项目推进(用户不是来交付代码的)、用通用 PM 而非 AI PM 视角评估、加入与"找实习"无关的工程优化(性能、可访问性、单元测试等)。

## 先看哪两份文档

仓库根目录有两份**针对作品集状态**的文档,**优先于 PRD 阅读**:

- **`AI_PM_DIAGNOSIS.md`** — 五维度诊断(产品深度 / AI 含量 / 设计完整度 / 商业 sense / 可讲性)、强项、短板、三个面试 talking points、一个空洞。**做任何"该不该补 X"的判断,先看这份。**
- **`AI_PM_ROADMAP.md`** — 基于诊断的补救清单,按 ROI 排了 P0/P1/P2。**做任何"下一步做什么"的判断,先看这份。**

如果用户来问"我接下来做什么 / 这事值不值得做",**默认动作是先 Read 这两份再回答**,不要让用户重复说背景。

## 仓库里有什么

不是可运行代码,没有 build/lint/test 命令——所有 HTML 直接用浏览器打开。

两类产物:

1. **PRD**:`part_time_wechat_mini_program_project_mvp.html`(单文件,~1300 行)。中文,四方业务闭环(应聘方 / 平台 / 招聘方 / 中介方)的兼职小程序 MVP 规格。**这是用户产品基本功的主要载体**,§3 关键设计决策、§5 状态机、附录 B v2→v3 修订表都是可直接用于面试的素材。
2. **设计原型**:`model_selection_business/*.html`。"中介选模"子功能的 iPhone 尺寸交互原型。React 18 走 unpkg UMD 加载,**没有 JSX**(组件以预编译的 `React.createElement` 调用形式书写)。
3. **外置 SVG**:`diagram-*.svg`,是 PRD 内嵌图的副本(同步关系见下面"编辑禁区")。

## 编辑禁区(短)

下列点不是 `ls` 一眼看得出来,踩坏了用户要回头修很久——做任何文件改动前先确认没违反:

- **PRD 编号交叉引用是承载性的**:消息编号(#1–#22, #19a)、状态名、超时编号(T1–T5, T11)、异常 alt 分支(alt 1–4)、HTML 锚点(`#sec-3` 等)被多节加附录 A 三向索引引用。**不要重新编号**。
- **§3 两条"故意的不对称"是设计意图,不是 bug**:
  - T1/T3 系统替用户决策,T2 故意**不**替招聘方做"默认通过"(§3.4)
  - MVP 取消统一处理、**不分级退款**(§3.3),分级补偿是 v2 项
  把这两条"改成一致"等于违反规格。
- **`diagram-*.svg` 与 PRD 内嵌 `<svg>` 块是双份的**:改一处必须改两处。
- **原型用 `React.createElement`,不是 JSX**:别往里塞 JSX 语法。
- **跨 `<script>` 块的桥靠 `window` 全局**:`Object.assign(window, {...})` 导出不能删。
- **全中文**:UI 文案、PRD 正文、领域术语("招募令"、"递补"、"异议重筛")不要翻译成英文,不要换同义词。
- **`casting-prototype-standalone.html`**(~2.9 MB)是不透明 base64 打包件,**不要手工编辑**。
- **末尾 `<style id="__om-edit-overrides">`** 是宿主生成的,不要动。

## 工作时的默认动作

- **回答"这件事值不值得做"时**:先读 `AI_PM_DIAGNOSIS.md` + `AI_PM_ROADMAP.md`,对照里面的 P0/P1/P2 给意见;不要让用户重复讲"我是大三找 AI PM 实习"。
- **新增任何文档**:默认中文(memory `feedback-language-chinese`),并在 `AI_PM_ROADMAP.md` 里加一条更新记录(让路线图始终是最新状态)。
- **如果用户的请求与 ROADMAP 冲突**(比如想做 P2 却没做 P0):直接说出来,问他是不是要调整优先级,而不是默默执行。
- **保持诚实**:这是作品集,**夸出来的强项在面试时会被戳穿**。诊断里"短板"那部分如果发现新的问题,直接更新进去,不要因为"已经写过了"就回避。

## 怎么"运行"

任意 HTML 直接拖到浏览器即可。`casting-prototype-standalone.html` 需要支持 `DecompressionStream` 的现代浏览器。
