# 四方协作兼职小程序 · 产品作品集

> 作者:大三在校,产品方向 **AI 产品经理**。本仓库是一份产品作品集——不是要上线的工程代码,而是用一个真实业务问题展示产品规格能力、AI 落地判断与数据思考。

## 这是什么

一个「应聘方 ↔ 中介方 ↔ 招聘方 ↔ 平台」四方协作的兼职撮合小程序。核心产品工作集中在三件事:

1. **业务闭环规格**:四方角色、主流程、状态机、异常路径、v2→v3 迭代修订。
2. **AI 落地判断**:MVP 故意**不上 AI**、全流程人工跑通;AI 作为 **v2 辅助筛选**(人机协同,不是取代中介)单独成规格。
3. **数据思考**:每条关键设计决策配验证指标与可证伪逻辑。

## 关于 AI(主动说清楚)

这是 AI PM 作品,但 MVP **刻意没有 AI**:冷启动期用中介方人工筛选跑通四方信任、避免 AI 假阳性破坏闭环、同时让人工筛选沉淀出 v2 的训练样本。AI 在哪、为什么 v1 不做 v2 才做、怎么按数据成熟度演进——完整论证见 `AI_MATCHING_SPEC.md`,PRD §3.5 也有三段式决策记录(均标注 `v2·本版不做`)。
